City Stress Index

A multi-source urban intelligence platform that scores and ranks 5 global cities — Toronto, New York, London, Tokyo, Mumbai — across five real-world stress dimensions, collapsing them into a single composite City Stress Index.

Engineered modular Python collectors against the TomTom Traffic Flow API — capturing congestion indices, free-flow speeds, and road segment metrics across 50+ monitored corridors — alongside Numbeo API feeds for cost-of-living indicators and crime indices, and weather severity scoring, all piped into automated timestamped JSON archives with retry logic and error handling.

Designed a production-grade SQLite star schema — 1 central Traffic Fact table, 4 dimension tables (City, Road Segment, Date, Time) — ingesting multi-source raw data via a Python ETL pipeline (Pandas, SQLAlchemy), improving cross-city query performance by 40% over flat-file processing. Phase 5 feature engineering computes 15+ derived metrics: normalised index scores, rolling 7-day averages, volatility bands, peak-hour spike flags, trend indicators, and a weighted composite stress score across all five dimensions.

Next phase: Prophet forecasting models targeting MAPE < 10% across all 5 city corridors, followed by a 4-page interactive Power BI dashboard with city heatmaps, anomaly detection alerts, forecast vs. actual overlays, and cross-city benchmarking panels.

Retail Sales Forecasting
& Demand Planning

Ingested and cleaned 1M+ transactional records spanning 2 years using Python (Pandas, NumPy) — resolving 34K duplicates, 19K cancellations, and 243K missing customer IDs across 43 countries and 4,735 products, enforcing completeness, accuracy, and schema consistency throughout.

Built and benchmarked SARIMA and Holt-Winters (Statsmodels) forecasting models on monthly revenue data using a chronological 80/20 train-test split. SARIMA achieved a MAPE of 25.54% (74.5% accuracy) — with model evaluation documented across MAE, RMSE, and AIC for rigorous comparison. Designed a SQL star schema warehouse (SQLite) with 1 fact table and 4 dimension tables, loading the full 1M+ record dataset via a Python ETL pipeline (SQLAlchemy).

Built a 4-page Power BI dashboard — dynamic KPI cards, seasonal revenue trend lines, actual vs. forecast overlays with 80% confidence intervals, and inventory risk flags with traffic-light formatting. Applied RFM scoring to segment 5,861 customers across 6 tiers: identifying 843 Champions averaging £5,689 spend, and 409 At Risk customers flagged as the highest-priority re-engagement target for retention campaigns.
